# pfm_app ドキュメント目次

設計・方針をここに外出しして「コンテキスト分散」する。新しいチャット/セッションでは、必要なファイルだけ読めば文脈を復元できる。

| # | ファイル | 中身 | いつ読む |
|---|---|---|---|
| 01 | [overview.md](./01-overview.md) | プロジェクトの目的・背景・対象企業 | 全体像を思い出したい時 |
| 02 | [architecture.md](./02-architecture.md) | 技術構成・全体図・採用理由 | 技術選定の根拠を確認する時 |
| 03 | [roadmap.md](./03-roadmap.md) | イテレーション計画・未決事項 | 次に何を作るか決める時 |
| 04 | [domain-ledger.md](./04-domain-ledger.md) | 複式簿記の台帳ドメイン設計 | モデルを書く/レビューする時 |
| 05 | [how-we-work.md](./05-how-we-work.md) | 学習方針(ストイック)・役割分担・トークン戦略 | 進め方に迷った時 |
| — | [notes/](./notes/) | イテレーションごとの作業ログ（設計判断/詰まった点/次） | 振り返り・経緯確認 |

> ルール: 設計判断が変わったら**まずここのmdを直す**。チャットの会話は揮発するが、mdは残る。
